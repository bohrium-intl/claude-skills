## 1. Design Philosophy

PhdPal social visuals should feel like a **brand statement or thought piece**, not a tutorial or how-to guide.

### The Mindset
- You are making a **product teaser**, not a lesson plan
- Every slide should make someone **curious about PhdPal**, not teach them how to use it
- Think Monocle magazine meets academic authority — editorial, confident, spare

### Core Principles
| Principle | What It Means |
|-----------|--------------|
| **Editorial, not instructional** | No "Step 01 / Step 02 / Step 03" labels. No numbered how-to lists. Show the system, don't teach it. |
| **Emotional, not informational** | Pain points should feel like "this is YOUR pain" — not "here are 3 issues". Use second person ("You wrote 5,000 words..."), not third person. |
| **Left-aligned, not centered** | Center-align is lazy. Left-aligned text with asymmetric layouts feels more intentionally designed. Exception: quote slides can center. |
| **Rhythm through contrast** | Alternate dark/light backgrounds across slides. Never put 2+ light slides or 2+ dark slides in a row. |
| **One accent per slide** | Each slide gets one color accent moment. Don't scatter accents everywhere. |

## 2. Brand Color System

### Dark Slides
- Background: brand gradient `linear-gradient(194deg, #A5ACFF 5%, #5F64FF 100%)` or deep indigo `#2E2A6E`
- Text: `#FFFFFF` (headings), `rgba(255,255,255,0.5)` (body), `rgba(255,255,255,0.2)` (footer)
- Accent text: `#27D0FF` (cyan) for emphasis words
- Accent stripe: `#6A29FF` (vivid purple) or `#27D0FF` (cyan)

### Light Slides
- Background: `#F0F0FD` (brand cream)
- Text: `#1D2129` (headings), `#4E5969` (body), `#86909C` (muted)
- Accent: `#5F64FF` (indigo) for kickers, active states, borders
- Active/highlight fill: `#5F64FF` with white text inside

### Geometric Decoration (from PhdPal logo)
Use **sharp geometric shapes** as background decoration — NOT blurry gradient blobs:
- **Circles**: `border-radius: 50%`, brand colors at 8-20% opacity
- **Triangles**: CSS borders (`border-left/right: transparent, border-bottom: solid`), 4-12% opacity
- Colors: `rgba(39, 208, 255, 0.15)` (cyan), `rgba(106, 41, 255, 0.2)` (purple), `rgba(255,255,255,0.05)` (white on dark)
- Place 2-3 per slide, partially off-screen for organic feel

### Accent Stripe
Every slide gets a **10px vertical stripe** on the left edge (`position: absolute; left: 0; top: 0; bottom: 0; width: 10px`).
- Vary the color per slide: vivid purple → cyan → indigo → vivid purple
- This creates a "series" feel across the carousel

## 3. Typography

### Fonts
- **Headings**: `'Plus Jakarta Sans', sans-serif` — weight 800, letter-spacing -0.02em
- **Body**: `'Inter', sans-serif` — weight 400-600
- **Kickers/labels**: `'Plus Jakarta Sans'` or `'Inter'` — weight 700, letter-spacing 4px, uppercase, 16px

### Size Scale (for 1080x1350 slides, optimized for X readability)
| Element | Size | Weight |
|---------|------|--------|
| H1 (cover title) | 96px | 800 |
| H2 (section title) | 64-72px | 800 |
| Body / pain points | 30-34px | 600-700 |
| Descriptions | 20-24px | 400 |
| Kickers (uppercase labels) | 16px | 700 |
| Footer / brand marks | 16px | 500 |

**Critical**: These sizes are calibrated for X where images get compressed. Regular web sizes (14-18px) will be unreadable. When in doubt, go bigger.

### Emphasis Pattern
Use `<em>` tags for key phrases. Style them:
- On dark backgrounds: `color: #27D0FF` (cyan), `font-style: normal`
- On light backgrounds: `color: #5F64FF` (indigo), `font-style: normal`
- Never use italic — sans-serif italic looks weak at large sizes

## 4. Slide Composition Patterns

### Carousel Flow (4-slide standard)
```
Slide 1: HOOK (brand gradient bg)
  → Provocative statement, not a description
  → Large title, kicker above, subtitle below
  → Left-aligned

Slide 2: PAIN (dark bg — #2E2A6E or similar)
  → Emotional pain points in second person
  → Dashes (—) not numbers
  → Faded secondary text for contrast: "You did X" <span style="opacity:0.4">but Y happened</span>

Slide 3: SYSTEM (light bg — #F0F0FD)
  → Show the framework as a visual system, not a tutorial
  → Timeline/vertical flow with one highlighted "active" item
  → Active item: filled background (#5F64FF), white text, box-shadow glow

Slide 4: TAKEAWAY (brand gradient bg)
  → Bold quote or statement
  → Large quotation mark as watermark (360px, 8% opacity)
  → Corner marks (L-shaped borders) for editorial framing
```

### Layout Rules
- **Padding**: 72px top/bottom, 80px left/right (for 1080px wide slides)
- **Footer**: Brand name at bottom, separated by a thin rule, 16px uppercase
- **z-index**: Always set `position: relative; z-index: 1` on content that sits above geometric decorations
- **Dividers**: 1px solid rules — `rgba(255,255,255,0.08)` on dark, `#D0CEE8` on light. Never use thick decorative dividers.

## 5. Anti-Patterns (Things That Make It Look Like AI)

| Don't | Why | Do Instead |
|-------|-----|------------|
| Numbered lists (01, 02, 03) | Looks like a tutorial/infographic | Use dashes (—) or no markers |
| "Step X" labels | Instructional, not editorial | Just show the flow visually |
| Glassmorphism cards everywhere | 2024 AI fingerprint | Use thin rules, accent fills, whitespace |
| Gradient text | Decorative, not meaningful | Use solid color with one accent color for emphasis |
| Blur blobs for decoration | Generic, no brand identity | Use sharp geometric shapes from logo (circles, triangles) |
| Same background on all slides | Monotonous, no scroll-stop | Alternate dark/light for rhythm |
| Centered everything | Lazy, generic | Left-align text, asymmetric compositions |
| Small text (< 20px body) | Unreadable on X/mobile | Min 20px for body, 64px+ for titles |
| "Point One / Point Two / Point Three" | Placeholder energy | Use concrete words (Data, Logic, Cases) |

## 6. Export Workflow

### HTML → 2x PNG via Puppeteer

After creating the HTML slides, export them as individual 2x PNGs:

```javascript
const puppeteer = require('puppeteer');
const path = require('path');

(async () => {
  const browser = await puppeteer.launch({
    executablePath: '/Applications/Google Chrome.app/Contents/MacOS/Google Chrome',
    headless: true,
  });
  const page = await browser.newPage();
  await page.setViewport({ width: 1200, height: 1500, deviceScaleFactor: 2 });

  const filePath = path.resolve(__dirname, 'slides.html');
  await page.goto('file://' + filePath, { waitUntil: 'networkidle0' });
  await page.evaluate(() => document.fonts.ready);
  await new Promise(r => setTimeout(r, 1000));

  const slides = await page.$$('.slide');
  for (let i = 0; i < slides.length; i++) {
    await slides[i].screenshot({
      path: path.resolve(__dirname, `slide-${i + 1}.png`),
      type: 'png'
    });
  }
  await browser.close();
})();
```

### Platform Sizes
| Platform | Slide Size (CSS) | Export (2x) |
|----------|-----------------|-------------|
| X carousel | 1080 × 1350 | 2160 × 2700 |
| X single image | 1200 × 675 | 2400 × 1350 |
| LinkedIn | 1200 × 627 | 2400 × 1254 |
| Instagram | 1080 × 1080 | 2160 × 2160 |

### Checklist Before Export
- [ ] All text readable at phone-screen size?
- [ ] Dark/light rhythm across slides?
- [ ] No two adjacent slides with same background?
- [ ] Accent stripe on every slide?
- [ ] Geometric shapes partially off-screen, not floating in center?
- [ ] No tutorial markers (Step X, numbered lists)?
- [ ] Footer/brand mark on every slide?
- [ ] Fonts loaded (wait for `document.fonts.ready`)?

## 7. Quick Start

When the user asks to create PhdPal social visuals:

1. **Clarify**: What platform? How many slides? What's the message?
2. **Invoke** `frontend-design` skill for general aesthetics guidance
3. **Compose** slides following the carousel flow pattern above
4. **Apply** brand colors from section 2 (or cross-reference the Social Media Guide)
5. **Review** against the anti-patterns table
6. **Export** via Puppeteer at 2x, copy PNGs to Desktop
